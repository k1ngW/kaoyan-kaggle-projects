## 1. 这个比赛在做什么

### 问题

给你 57 万颗天体的观测数据（亮度、颜色、红移、天球坐标），判断每颗天体属于三类中的哪一类：

| 类别 | 中文 | 特征 |
|------|------|------|
| **GALAXY** | 星系 | 延展源，颜色偏红，红移分布广 |
| **QSO** | 类星体 | 点源，颜色偏蓝，紫外出超 |
| **STAR** | 恒星 | 点源，颜色在中间，红移接近 0 |

### 数据长什么样

```
一颗天体的数据:
  alpha=150.2  delta=2.3      ← 天球坐标
  u=19.5  g=18.2  r=17.1  i=16.8  z=16.5    ← 5个波段的星等
  redshift=0.12                              ← 红移
  class=GALAXY                                ← 我们要预测的
```

星等越小越亮，不同波段的差值 = 颜色 = 天体类型的关键信号。

### 评估指标：Balanced Accuracy

```
普通的 Accuracy = 正确数 / 总数
→ 65% 是 GALAXY，全猜 GALAXY 就能 65%，但对模型没用

Balanced Accuracy = (GALAXY召回率 + QSO召回率 + STAR召回率) / 3
→ 每类平等对待，必须三类都猜对
```

## 2. 特征工程：模型看到的世界

### 原始特征不够用

模型看到 `u=19.5, g=18.2` 这样两个数字，它不知道"u 和 g 的差值"才是关键。

### 你项目里实际做的特征（v8 体系）

**颜色差（最重要）**
```python
u_g = u - g   # u波段比g波段暗1.3等 → 偏红
g_r = g - r
r_i = r - i
i_z = i - z
```
天文学常识：恒星在 `u-g` vs `g-r` 图上形成特定轨迹，星系和类星体在不同位置。这 10 对颜色差是区分三类最核心的信息。

**统计特征**
```python
mag_mean = (u+g+r+i+z)/5     # 平均亮度
mag_std  = std(u,g,r,i,z)    # 亮度变化
mag_min/max/range             # 最亮/最暗/范围
```

**红移交互**
```python
redshift_u = redshift × u    # 红移和亮度的交互
u / redshift                  # 比值
```
类星体的红移通常很大且和亮度有特定关系，星系则不然。

**目标编码 (Target Encoding)**
```python
# 对每个分桶后的特征值，统计它对应三类目标的概率
TE_redshift_qbin64_GALAXY = 红移第X号桶 → GALAXY概率
```
这是 Chris Deotte 方法的核心：把连续变量分桶，然后用"这个桶里有百分之几是 GALAXY/QSO/STAR"作为特征，比原始数值更直接。

**为什么不直接给模型原始数据？** 因为模型（特别是树模型）不擅长自动发现 `u-g` 或 `u/g` 这样的关系。你帮它算好，它就能专注学"什么颜色区间对应什么天体"。

## 3. 交叉验证：怎么知道模型好不好

### 问题：不能拿测试集验证

测试集的标签是隐藏的，提交了才知道分数。如果每次调参都提交一次，不仅浪费提交机会，而且 LB 有噪音（后面会讲）。

### 5 折交叉验证 (5-Fold CV)

```
把训练集随机分成 5 份:

Fold 1: [训训训训验]
Fold 2: [训训训验训]
Fold 3: [训训验训训]
Fold 4: [训验训训训]
Fold 5: [验训训训训]

每次用 4 份训练，1 份验证
5 次验证分数取平均 = CV score
```

### StratifiedKFold

普通随机分可能某个 fold 里全是 GALAXY。Stratified 保证每个 fold 里的类别比例和整体一致（都是 65% GALAXY / 18% QSO / 17% STAR）。

### OOF (Out-Of-Fold) 预测

```
每个样本在被当作"验证集"的那一次，模型没见过它
→ 那次预测 = 该样本的 OOF 预测

所有样本的 OOF 拼起来 → oof.npy  (shape: N×3)
```

**OOF 的意义**：模拟了模型对"没见过的新数据"的表现，比训练集上的分数更接近真实 LB。

### CV 和 LB 的噪音

Chris 帖子里原话：**LB score is pretty noisy**。CV 提升 0.0003 不代表 LB 一定提升，CV 下降不一定 LB 下降。

为什么？测试集只有 ~25 万样本，随机波动叠加类别不平衡，小差距不可靠。**以 CV 为主，LB 为辅做决策**。

## 4. XGBoost：一棵树一棵树地学习

### 树是什么

想象你要区分 GALAXY 和 STAR，你画一个决策流程：

```
u - g <= 0.8 ?
  ├── 是 → r - i <= 0.3 ?
  │         ├── 是 → GALAXY
  │         └── 否 → STAR
  └── 否 → redshift > 0.5 ?
            ├── 是 → QSO
            └── 否 → GALAXY
```

一棵决策树就是把数据按特征条件反复切分，直到每个叶子上基本是同一类。

### Boosting：一棵接一棵地修正

XGBoost 的核心思想：

```
第1棵树: 做一个粗糙的分类 → 预测
         发现有些样本猜错了 → 计算"残差"（错误的大小）

第2棵树: 专门去拟合残差 → 修正第1棵树的错误
         这棵树不太准也没关系，只加一小部分（learning_rate=0.05）

第3棵树: 继续拟合新残差
...

第7000棵树: 每次加一点点，7000 棵小树加起来就很强了
```

类比：学射箭。第一次偏左 10cm，第二次修正 5cm，第三次修正 2.5cm... 每次修正量递减，但最终越来越准。

### 关键参数

| 参数 | 你的 v8 值 | 含义 |
|------|:--:|------|
| n_estimators | 7000 | 总共 7000 棵树 |
| learning_rate | 0.012 | 每棵树只贡献 1.2% |
| max_leaves | 72 | 每棵树最多 72 个叶子 |
| max_depth | 0 | 不限制深度（lossguide 模式） |
| subsample | 0.82 | 每棵树随机用 82% 的数据 |
| colsample_bytree | 0.74 | 每棵树随机用 74% 的特征 |
| early_stopping_rounds | 180 | 180 轮不提升就停 |

**learning_rate 小 + n_estimators 大** = 慢慢学，不怕过拟合。**subsample + colsample** = 每棵树看到的都不一样 → 多样性 → 泛化好。

### XGBoost 的局限

- 只能学特征空间的"轴对齐"分割（沿坐标轴切，不能斜着切）
- 对类别特征需要手动编码（onehot 或 label encode）
- 不会自动发现 `u-g` 这种交互 → 需要你帮它算好

## 5. CatBoost：XGBoost 的升级版

### 和 XGBoost 的核心区别

| | XGBoost | CatBoost |
|------|------|------|
| 树的分裂方式 | 按特征值阈值切 | 同，但多了类别特征原生支持 |
| 处理类别特征 | 需要先转成数字 | **原生支持**，自动做最优编码 |
| 梯度计算 | 用全量数据算梯度 | **Ordered Boosting**：用历史数据算梯度 |
| 特征交互 | 单调分裂 | **CTR3**：自动组合类别特征 |

### Ordered Boosting

XGBoost 的一个问题：用同一批数据算梯度再预测 → 信息泄露 → CV 和 LB 差距大。

CatBoost 的解法：对于第 i 个样本，只用前 i-1 个样本来算梯度。这样每个样本的梯度计算"没看到过自己"，CV 更接近 LB。

### 原生 CTR3（类别特征交互）

你的 v17_cat 用了 153 个类别特征。CatBoost 会自动尝试"哪些类别特征的组合能提升预测"，不用你手动构造 `spectral_type × galaxy_population` 这种交互。

### 为什么 CatBoost 在你的比赛里最强

```
类别特征多（153个） → CatBoost 原生支持 → 自动发现有效组合
Ordered Boosting     → CV/LB 更一致      → 不容易过拟合
GPU 原生             → T4 上跑得快        → 调参迭代更多
```

## 6. RealMLP：用神经网络找规律

### 神经网络是什么

回到树模型：树是"如果 u-g>0.8 且 r-i<0.3 则 GALAXY"这种规则。

神经网络不学规则，它学**变换**：

```
输入 [u, g, r, i, z, redshift, ...]  ← 171 个特征
   ↓ ×W1 + 非线性
隐藏层1: [128 个神经元]
   ↓ ×W2 + 非线性
隐藏层2: [256 个神经元]
   ↓ ×W3 + 非线性
...
   ↓ ×W_last + softmax
输出: [GALAXY概率, QSO概率, STAR概率]
```

每个神经元是一个简单的"加权求和 + 激活"，几百个神经元叠起来就能表达非常复杂的边界。

### 和树模型的区别

| | 树模型 | 神经网络 |
|------|------|------|
| 决策边界 | 轴对齐的方块 | **任意曲线** |
| 特征交互 | 需要手动构造 | **自动学** u×g 这种关系 |
| 训练方式 | 逐棵树加 | 反向传播梯度下降 |
| 过拟合倾向 | 低（每棵小树贡献小） | **高**，需要正则化 |

### RealMLP 特殊在哪

普通的 MLP（多层感知机）处理表格数据表现一般。RealMLP 专门为表格数据做了优化：

1. **数值嵌入**：不是直接把 u=19.5 喂进去，而是先做 piecewise-linear encoding（切段后线性编码），和 XGB 的分桶逻辑类似
2. **Robust 缩放**：自动处理异常值和不同量纲
3. **8-ensemble 内部投票**：一个 RealMLP 模型 = 8 个子网络同时训，取平均

### 你的 v18 为什么这么强

```
RealMLP-5 = 5-fold × 8-ensemble = 40 个子网络
每个子网络结构不同但共享训练逻辑
→ 内部已经在做"小规模集成"
→ 单模 CV 0.96904，冠绝所有单模
```

## 7. TabM：用注意力机制看表格

### Attention 是什么

传统神经网络：每个特征同等对待，通过权重矩阵慢慢学哪个重要。

Attention：让模型**自己学"看到特征 A 时应该多关注特征 B"**。

类比：你在一张星图里找星系，你不会逐个像素平等扫视。你的眼睛会**注意**到"这个区域颜色和周围不同"，然后聚焦过去。注意力机制就是这个过程。

### TabM 的训练方式：One-vs-Rest

你的 v20 notebook 没做传统三分类，而是：

```
子任务1: 这颗天体是 QSO 还是"不是QSO"？ → 输出 [是QSO的概率]
子任务2: 这颗天体是 STAR 还是"不是STAR"？→ 输出 [是STAR的概率]
子任务3: 这颗天体是 GALAXY 还是"不是GALAXY"？→ 输出 [是GALAXY的概率]

三个概率拼起来 → 归一化 → 三分类
```

为什么这么做？因为 QSO 的特征模式（紫外出超 + 高红移）和其他两类差异巨大，专门训一个"QSO 检测器"比三分类更聚焦。

### TabM vs RealMLP

| | RealMLP | TabM |
|------|------|------|
| 核心机制 | 多层全连接 + 数值嵌入 | **Self-Attention** 跨特征交互 |
| 学习方式 | 前馈 | 特征间两两交互 |
| 训练 | 多分类 | One-vs-Rest 二分类 |
| 优势 | 稳定，CV 最高 | 能学到"特征 A 和特征 B 同时出现"的模式 |

### 为什么 TabM + RealMLP 互补

RealMLP 学的是"特征组合 → 类别"的全局映射。

TabM 的 Attention 学的是"对于这个样本，哪几个特征之间最有关系"。比如：

```
样本X: TabM 可能注意到 u-g 和 redshift 高度相关 → 这是 QSO 的信号
       而 RealMLP 可能通过 g-r 和 r-i 的组合来判断

两个模型抓住了不同的信号 → 集成互补
```

## 8. 集成：三个臭皮匠顶个诸葛亮

### 为什么集成有效

假设三个模型在 100 个测试样本上的预测：

```
样本1: 模型A(错)  模型B(对)  模型C(对) → 投票后(对)
样本2: 模型A(对)  模型B(对)  模型C(错) → 投票后(对)
样本3: 模型A(对)  模型B(错)  模型C(对) → 投票后(对)
```

每个模型单独正确率 67%，但**它们错的样本不一样**。投票后正确率接近 100%。

关键前提：**模型之间要犯不同的错误**。如果三个模型犯的错误完全一样，集成没收益。

### 架构多样性的价值

```
v19 实验:
  树 + 树 (cat + xgb)     → 提升微弱    ← 犯同样的错误
  树 + NN  (cat + realmlp) → 提升显著    ← 犯不同的错误

v20 预期:
  GBDT + MLP + Attention = 三种不同的"思考方式"
  → 集成的边际收益最大
```

### 简单加权平均（v19 用的）

```
对每个测试样本:
  blend概率 = 0.47 × realmlp概率 + 0.48 × cat概率 + 0.05 × v8概率
  预测 = argmax(blend概率)  ← 谁概率最高选谁
```

权重是通过 Grid Search 搜出来的：在 OOF 上试几万种权重组合，看哪种 CV 最高。CV 最高 ≠ LB 最高，但通常是可靠方向。

### LogisticRegression Stacking（v20 notebook 用的）

```
不用固定的 w1, w2, w3，而是训练一个逻辑回归:
  LogReg 输入: [model1的预测, model2的预测, model3的预测]
  LogReg 输出: 融合后的预测
  LogReg 自动学到: "QSO 类多听 TabM 的，STAR 类多听 RealMLP 的"
```

比简单加权平均灵活，能学到**非线性**和**类间差异**。

## 9. 后处理优化：不动模型的提升空间

### 为什么后处理值得做

```
原始模型         → 给你概率 [GALAXY=0.33, QSO=0.34, STAR=0.33]
argmax          → 选 QSO（赢了一点点）
但 QSO 实际上只有 18%，阈值天然偏低
调整后 argmax   → 考虑到 QSO 先验概率低，可能需要更高的 QSO 概率才选它
```

后处理 = 在模型输出的概率上做调整，不重新训练。

---

### 方案 1：Grid Search 固定权重（基线）

```
融合概率 = w1 × P_v8 + w2 × P_cat + w3 × P_realmlp + w4 × P_tabm
Grid Search: 所有 (w1,w2,w3,w4) 组合 → 选 CV 最高的
```

**什么时候有用**：4 个模型质量差不多，简单平均就很接近最优解。

**局限**：每个模型只有一个全局权重。TabM 在 QSO 上强、在 STAR 上弱，全局权重只能折中。

---

### 方案 2：12 权重（每模型 × 每类）

```
GALAXY 概率 = w_v8_G × P_v8_G + w_cat_G × P_cat_G + w_realmlp_G × P_realmlp_G + w_tabm_G × P_tabm_G
QSO 概率    = w_v8_Q × P_v8_Q + ...
STAR 概率   = w_v8_S × P_v8_S + ...

每个类的 4 个权重独立搜索
```

**为什么比方案 1 好**：Cell 28 已经证明了——QSO 通道 TabM 独占（系数+16），外部模型被压到 0。如果只给 TabM 一个全局权重，它在 STAR 和 GALAXY 上也被迫用同样权重，没法"QSO 上打满，STAR 上少用点"。

**什么时候有用**：模型有明显的"偏科"（某个模型在某个类上特别强）。

---

### 方案 3：XGB Stacking（非线性融合）

```
简单加权 = 四个模型概率的线性组合
XGB Stacking = 训练一个 XGB，让它学"什么情况下该信哪个模型"

XGB 能学到:
  "如果 TabM 的 QSO 概率 > 0.95 且 RealMLP 的 QSO 概率 < 0.01 → 还是选 QSO"
  "如果 4 个模型给的概率差不多 → 选 RealMLP 的（因为它最稳定）"
```

**为什么 XGB 而不是 LogReg**：LogReg 是线性的——它只能学 `w1×P1 + w2×P2`。XGB 能学 `if P1>0.8 and P2<0.3 then ...` 这种非线性规则。

**过拟合风险**：OOF 数据量小（只有 4 个模型 × 3 类 = 12 个特征），XGB 树深了容易记住噪音。需要 `max_depth=3~5` 严格控制。

---

### 方案 4：阈值调优

```
当前: 融合概率 → argmax → 预测类别

阈值调优后:
  融合概率 × [t_galaxy, t_qso, t_star] → argmax → 预测类别

t_galaxy > 1.0 → 需要更高的概率才选 GALAXY
t_qso < 1.0    → 更容易选 QSO（降低门槛）
```

**什么时候有用**：某类系统性地被过度预测或欠预测。比如模型总把边界样本判为 GALAXY（因为训练集里 GALAXY 最多），提高 GALAXY 阈值可以纠偏。

**scipy.minimize 自动搜**：不需要手动试，给目标函数（最大化 CV），scipy 自己找最优的 3 个 t 值。

**局限**：没改变概率值的大小关系，只是缩放。如果概率本身就是偏的，调阈值能缓解但不能根治。

---

### 方案 5：每类独立 XGB Stacking（终极方案）

```
方案 2 + 方案 3 的结合:

QSO 通道:    XGB(4个模型的QSO概率) → 融合后的 QSO 概率
STAR 通道:   XGB(4个模型的STAR概率) → 融合后的 STAR 概率
GALAXY 通道: XGB(4个模型的GALAXY概率) → 融合后的 GALAXY 概率
```

和 v20 notebook 的 `run_blend()` 同思路，但 meta model 从 LogReg 升级为 XGB。

**为什么最强**：三个类完全解耦，每个类有自己的专家模型。XGB 能学非线性，且每类独立学不容易被多数类主导。

**风险**：最复杂，过拟合风险最高。需要严格用 OOF 做 5-fold stacking（不能在同一条数据上训和测）。

## 10. v23 实战：LogReg Stacker 经验教训

> 2026-06-15，基于 Chris Deotte 的 `gpu-logistic-regression-stacker` 方案，用 5 个基模型的 OOF 训练 Linear + L2 堆叠器。

### 10.1 核心思路

```
基模型 OOF (概率) → logit 变换 → 标准化 → Linear(N_channels→3) + CrossEntropy + L2
                                                      ↓
                                          5 seed × 5 fold = 25 次
                                                      ↓
                                          25 次预测取平均 → 最终概率
```

**为什么用 logit 而非 raw prob**：概率在 [0,1]，加性关系扭曲。logit 在 (-∞,+∞) 无限可分，线性模型能正常工作。

### 10.2 活权重 vs 死权重

| | v19 Grid Search | v21 CSV 混合 | **v23 LogReg Stacker** |
|---|---|---|---|
| 权重 | 3 个手搜的固定数 | 4 个手搜的固定数 | **45 个自动学的系数** |
| 适应样本差异 | 否 | 否 | **是** |
| 跨类差异化 | 否 | 尝试过但失败 | **每类不同的 Top-5 通道** |
| 可解释 | 3 个数 | 4 个数 | **权重矩阵 + 排名** |

### 10.3 blood learnings

**教训 1：class_weights 在 logit 空间是毒药。**

OOF logit 已是去偏概率——GALAXY 的 logit 天然高（因为先验占比 65%），给它加权 0.51 = 强行压一半 → CV 0.954。去掉 class_weights → CV 0.967。**logit 空间样本不均衡已被消解，再加权只会破坏概率结构。**

**教训 2：L2 自动处理共线性。**

v17 和 v22a 都是 CatBoost 架构，OOF 高度相关。L2 会在共线特征中随机压死一个（loss 几乎不变）。5 通道 < 5 有效通道 → 需要更多**不同架构**的基模型。

**教训 3：权重分析直接指导迭代。**

Cell 4 输出每类的 Top-5 通道排名，告诉你：
- 谁在 Stacker 里是 MVP（v8 XGBoost，三类全进 Top-5）
- 谁被 L2 压死了（v17，只出现 1 次）
- 哪类需要更多多样性（STAR bias=-0.74，被系统性低估）
- 该补谁的 seed、退役谁（v22a 直接退）

**教训 4：5 通道不够。**

5 模型 × 5 fold × 5 seed → CV 0.96713，不如最优单模 v18 (0.96904)。Chris 原版 19 模型 → 0.97028。通道数在 ~10 前边际收益递减，5 太少了。

### 10.4 LogReg vs OvR Stacker

| | v23 LogReg | v23.2 OvR (备选) |
|---|---|---|
| 策略 | 一个 Linear(3N→3) | 每类独立 Linear(N→1) |
| 优势 | 全局最优，类间平衡自动学 | 每类自由选模型组合 |
| 劣势 | 偏科模型权重被摊薄 | 三个头训练代价 ×3 |
| 何时切换 | v23.1 仍不超单模时 | |

## 关键概念速查

| 概念 | 一句话 |
|------|------|
| **OOF** | 模型在"没见过的训练数据"上的预测，模拟测试集表现 |
| **CV** | 5 折 OOF 的平均分数，比 LB 稳定 |
| **LB** | Kaggle Public Leaderboard，有噪音 |
| **Balanced Accuracy** | 三类召回率取平均，防止多数类主导 |
| **StratifiedKFold** | 分折时保持每折类别比例一致 |
| **树模型** | 沿特征轴反复切分，学 if-else 规则 |
| **神经网络** | 学特征的非线性变换和组合 |
| **Attention** | 让模型自选"当前应该关注哪些特征" |
| **集成** | 多个模型投票，前提是它们犯不同的错 |
| **后处理** | 不动模型，只在输出概率上优化 |
| **Stacker** | 用基模型 OOF 训练 meta-learner 学习如何融合 |
| **活权重 vs 死权重** | 死权重 = 固定系数平均；活权重 = meta model 自动学到每类每模型的最优权重 |
| **In-context learning** | 预训练 transformer 不更新参数，把训练数据当 context 直接推理 |
| **logit 变换** | prob→log(p/(1-p))，让概率在 (-∞,+∞) 线性可分 |
| **伪标签** | 用模型预测的高置信度样本作为额外训练数据，bootstrapping |
| **对抗验证** | 训练分类器判断样本来源，检测训练/测试分布偏移 |
